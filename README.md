# y

An experimental [MediaWiki](https://www.mediawiki.org/wiki/MediaWiki)-like engine, written in TypeScript and C++, with emphasis on performance, UI/UX, customizability & configurability and attention to detail.

# Why?

This question is at the heart of this project. I mean, just read it's name.

On a serious note, let's be honest - there is no reason for this to exist, at least in it's current state. I'm just interested in the idea, so *why* not bring it to life? Who knows, maybe this will grow into something beautiful in the future.

# Current state

**This project is currently in the *very* early stages of development**. It is not yet usable and the only reason for running this on your device would be research purposes. But if you do get an itch to run this - please, let me know how it goes, I would love to get some feedback.

While we currently do not have much to offer, we do have some ideas on what we would like to see.

Here's a non-exhaustive list of the major features we would like to have:

* Wiki engine. Something between MediaWiki and Confluence/Notion.
* File "archive". Uploading & downloading files, configuring access rights, maybe even creating short links for them?
* Full-featured user management - groups, rights & permissions, blocking, renaming, etc.
* App-wide logging & reporting (?).
* ...

Because this is not serious project, I do allow myself to add new ideas to this list. I am aware of [feature/scope creep](https://en.wikipedia.org/wiki/Feature_creep), don't wory ;).

Here's a list of what this project will definitely *not* be, even though it might look like that's what I'm going for:

* Jira-like project/time management software, or anything related to that.
* Anything code related (git, code review, etc.).

Basically, this is *not* a clone of [Phabricator](https://www.phacility.com/). RIP ;(
